Structure Rules
========================
15) SYSTEM_PIPELINE: Focus on inference pipeline, not data generation or training.

      connected, or composed.
      INCLUDE: new module types, novel module interconnections, new pipeline topologies,
        parameter-sharing architectures, novel input/output routing.
      EXCLUDE ALL of the following (they are NOT architectural elements):
        ✗ Training/optimization innovations (loss functions, reward sharing, RL strategies)
        ✗ Benchmark/dataset contributions
        ✗ Evaluation methodology innovations (new metrics, evaluation frameworks)
        ✗ Paradigm-level claims ("domain-agnostic capability", "first X approach")
        ✗ Data construction methods
      Place training innovations in model_details.training_details instead.

16) EVALUATION_HIGHLIGHTS: 2-3 top results. No Table/Figure numbers. No overlap with main_takeaways.
    - QUALIFY SUPERLATIVES: "up to +15%" must specify which model/benchmark.

    - KEY_RESULTS with >5 entries: group by result_type, add result_group_narrative before each group.
